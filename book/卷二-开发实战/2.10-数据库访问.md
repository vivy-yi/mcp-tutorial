### 2.2 连接管理的艺术

数据库连接是稀缺资源，必须精心管理：

```mermaid
flowchart TB
    subgraph "连接管理模型"
        A1[请求] --> B1{有可用连接?}
        B1 -->|是| C1[获取连接]
        B1 -->|否| D1{已达上限?}
        D1 -->|否| E1[等待]
        D1 -->|是| F1[返回错误]
        E1 --> B1
        C1 --> G1[执行查询]
        G1 --> H1[归还连接]
    end
```

**关键设计原则**：

| 原则 | 说明 | 实现 |
|------|------|------|
| 连接池 | 复用连接，减少开销 | 预创建 N 个连接 |
| 超时保护 | 防止无限等待 | 设置连接获取超时 |
| 自动归还 | 防止连接泄漏 | 使用上下文管理器 |
| 健康检查 | 确保连接可用 | 定期检测连接状态 |

### 2.3 查询结果的处理艺术

数据库返回的数据往往很复杂，MCP 需要对其进行智能处理：

```mermaid
flowchart LR
    subgraph "原始数据"
        A1[数据库返回]
    end

    subgraph "MCP 处理"
        B1[类型转换]
        B2[字段筛选]
        B3[分页处理]
        B4[格式化]
    end

    subgraph "AI 可用"
        C1[结构化 JSON]
    end

    A1 --> B1 --> B2 --> B3 --> B4 --> C1
```

**设计要点**：
- 自动转换数据类型（日期、二进制等）
- 只返回必要的字段
- 大结果集自动分页
- 保持返回结构一致性

## 四、实践指南：安全配置与最佳实践

### 4.1 SQL 注入防护

SQL 注入是数据库最大的安全威胁：

```mermaid
flowchart TB
    subgraph "SQL 注入攻击"
        A1["用户输入: '; DROP TABLE users; --"]
        A2[直接拼接]
        A3["最终 SQL: SELECT * FROM users WHERE name = ''; DROP TABLE users; --'"]
    end

    subgraph "参数化查询"
        B1["用户输入: '; DROP TABLE users; --"]
        B2["作为参数传递"]
        B3["SQL: SELECT * FROM users WHERE name = ?"]
        B4["参数: ['; DROP TABLE users; --']"]
    end
```

**防护最佳实践**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    SQL 注入防护清单                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✅ 必须做：                                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ □ 使用参数化查询                                         │   │
│  │ □ 验证输入数据类型                                       │   │
│  │ □ 限制数据库用户权限                                     │   │
│  │ □ 启用查询日志审计                                        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ❌ 禁止做：                                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ □ 直接拼接用户输入到 SQL                                  │   │
│  │ □ 使用动态 SQL（EXECUTE）                                │   │
│  │ □ 在查询中暴露数据库错误信息                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

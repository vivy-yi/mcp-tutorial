# 2.2 TypeScript MCP 框架

> TypeScript 是构建 MCP 服务器的另一大主流选择，尤其适合需要与现有 JavaScript/TypeScript 项目集成的场景。本章将详细介绍 TypeScript 环境下的 MCP 开发。

---

## TypeScript 环境概述

TypeScript 为 MCP 开发带来了独特的优势：静态类型检查可以显著减少运行时错误，IDE 的智能提示能大幅提升开发效率，而与前端/JavaScript 生态的天然兼容性则使得全栈项目可以使用统一的技术栈。

```
┌─────────────────────────────────────────────────────────────────┐
│                    TypeScript vs Python 对比                      │
├──────────────────────┬───────────────────────────────────────────┤
│       特性          │                   说明                      │
├──────────────────────┼───────────────────────────────────────────┤
│       类型系统       │ 静态类型检查，编译时发现错误               │
├──────────────────────┼───────────────────────────────────────────┤
│       生态集成       │ 与 Node.js、前端框架无缝集成               │
├──────────────────────┼───────────────────────────────────────────┤
│       部署           │ 需要构建步骤，生成 JavaScript             │
├──────────────────────┼───────────────────────────────────────────┤
│       性能           │ V8 引擎优化，执行效率高                   │
├──────────────────────┼───────────────────────────────────────────┤
│       开发体验       │ 优秀的 IDE 支持和自动补全                  │
└──────────────────────┴───────────────────────────────────────────┘
```

确保你的开发环境已安装 Node.js 18 或更高版本：


```bash
# 安装 TypeScript
```

## 第一个 TypeScript MCP 服务器

### 基础示例

```typescript
// src/index.ts
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import {
  CallToolRequestSchema,
  ListResourcesRequestSchema,
  ListToolsRequestSchema,
} from '@modelcontextprotocol/sdk/types.js';

// 定义工具
const tools = [
  {
    name: 'greet',
    description: '向用户发送问候',
    inputSchema: {
      type: 'object',
      properties: {
        name: {
          type: 'string',
          description: '用户的名字'
        },
        greeting: {
          type: 'string',
          description: '问候语类型',
          enum: ['hello', 'hi', 'hey'],
          default: 'hello'
        }
      },
      required: ['name']
    }
  }
];

// 工具处理函数
async function handleToolCall(name: string, args: any) {
  switch (name) {
    case 'greet': {
      const { name, greeting = 'hello' } = args;
      const greetings: Record<string, string> = {
        hello: '你好',
        hi: '嗨',
        hey: '嘿'
      };
      return {
        content: [
          {
            type: 'text',
            text: `${greetings[greeting]}，${name}！欢迎使用 MCP！`
          }
        ]
      };
    }
    default:
      throw new Error(`Unknown tool: ${name}`);
  }
}

// 创建服务器实例
const server = new Server(
  {
    name: 'example-server',
    version: '1.0.0'
  },
  {
    capabilities: {
      tools: {}
    }
  }
);

// 注册工具列表处理器
server.setRequestHandler(ListToolsRequestSchema, async () => {
  return { tools };
});

// 注册工具调用处理器
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;
  return handleToolCall(name, args);
});

// 启动服务器
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('MCP Server started');
}

main().catch(console.error);
```

### 使用 SDK 简化开发

`@modelcontextprotocol/sdk` 提供了更高层次的抽象，简化了服务器开发：

```typescript
// src/index.ts - 使用 SDK
import { MCPServer } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { z } from 'zod';

// 创建服务器
const server = new MCPServer({
  name: 'my-server',
  version: '1.0.0'
});

// 定义工具 - 使用 Zod 进行类型验证
server.addTool({
  name: 'calculate',
  description: '执行数学运算',
  inputSchema: {
    a: z.number().describe('第一个数'),
    b: z.number().describe('第二个数'),
    operation: z.enum(['add', 'subtract', 'multiply', 'divide']).describe('运算类型')
  },
  handler: async ({ a, b, operation }) => {
    const operations = {
      add: a + b,
      subtract: a - b,
      multiply: a * b,
      divide: b !== 0 ? a / b : null
    };

    const result = operations[operation];

    if (result === null) {
      throw new Error('除数不能为零');
    }

    return {
      content: [
        {
          type: 'text',
          text: `结果: ${result}`
        }
      ]
    };
  }
});

// 定义资源
server.addResource({
  uri: 'config://app',
  name: 'App Config',
  description: '应用程序配置',
  mimeType: 'application/json',
  handler: async () => {
    return {
      contents: [
        {
          uri: 'config://app',
          mimeType: 'application/json',
          text: JSON.stringify({
            version: '1.0.0',
            environment: 'production'
          })
        }
      ]
    };
  }
});

// 启动
async function main() {
  const transport = new StdioServerTransport();
  await server.run(transport);
}

main().catch(console.error);
```

## 类型安全的工具定义

TypeScript 的核心优势在于类型安全。以下是如何充分利用这一优势：

### 定义工具接口

```typescript
// src/types/tools.ts
import { z } from 'zod';

// 定义输入参数
export const CreateUserInput = z.object({
  name: z.string().min(1).describe('用户姓名'),
  email: z.string().email().describe('用户邮箱'),
  age: z.number().min(0).max(150).optional().describe('用户年龄'),
  role: z.enum(['admin', 'user', 'guest']).default('user').describe('用户角色')
});

export type CreateUserInput = z.infer<typeof CreateUserInput>;

// 定义输出
export const CreateUserOutput = z.object({
  success: z.boolean(),
  userId: z.string(),
  data: CreateUserInput,
  createdAt: z.string().datetime()
});

export type CreateUserOutput = z.infer<typeof CreateUserOutput>;

// 工具定义中使用
export const createUserTool = {
  name: 'createUser',
  description: '创建新用户账号',
  inputSchema: {
    type: 'object' as const,
    properties: {
      name: { type: 'string', description: '用户姓名' },
      email: { type: 'string', description: '用户邮箱' },
      age: { type: 'number', description: '用户年龄' },
      role: { type: 'string', enum: ['admin', 'user', 'guest'], description: '用户角色' }
    },
    required: ['name', 'email']
  }
};
```

```typescript
    mimeType: 'application/json',
```

## 错误处理

### 工具级错误处理

```typescript
// src/utils/errorHandler.ts
import { CallToolResult } from '@modelcontextprotocol/sdk/types.js';

export class ToolError extends Error {
  constructor(
    message: string,
    public code: string,
    public statusCode: number = 400
  ) {
    super(message);
    this.name = 'ToolError';
  }
}

export function handleToolError(error: Error): CallToolResult {
  if (error instanceof ToolError) {
    return {
      content: [
        {
          type: 'text',
          text: JSON.stringify({
            error: error.code,
            message: error.message
          })
        }
      ],
      isError: true
    };
  }

  // 未知错误
  return {
    content: [
      {
        type: 'text',
        text: JSON.stringify({
          error: 'INTERNAL_ERROR',
          message: '发生内部错误',
          details: error.message
        })
      }
    ],
    isError: true
  };
}

// 在工具中使用
export async function safeDivide(a: number, b: number): Promise<CallToolResult> {
  try {
    if (b === 0) {
      throw new ToolError('除数不能为零', 'DIVISION_BY_ZERO', 400);
    }

    const result = a / b;

    return {
      content: [
        {
          type: 'text',
          text: `结果: ${result}`
        }
      ]
    };
  } catch (error) {
    return handleToolError(error as Error);
  }
}
```

## HTTP 传输层

TypeScript 版本同样支持 HTTP 传输，便于构建远程 MCP 服务：

```typescript
// src/serverHttp.ts
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { HttpServerTransport } from '@modelcontextprotocol/sdk/server/http.js';
import { SSEServerTransport } from '@modelcontextprotocol/sdk/server/sse.js';

const server = new Server(
  { name: 'http-server', version: '1.0.0' },
  { capabilities: { tools: {} } }
);

// 添加工具
server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: 'echo',
      description: '返回输入的内容',
      inputSchema: {
        type: 'object',
        properties: {
          message: { type: 'string', description: '要返回的消息' }
        },
        required: ['message']
      }
    }
  ]
}));

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  if (name === 'echo') {
    return {
      content: [{ type: 'text', text: args.message as string }]
    };
  }

  throw new Error(`Unknown tool: ${name}`);
});

// 启动 HTTP 服务器
async function main() {
  const transport = new HttpServerTransport({
    host: '0.0.0.0',
    port: 3000
  });

  await server.connect(transport);
  console.log('HTTP MCP Server running on http://localhost:3000');
}

main().catch(console.error);
```

## 测试 TypeScript MCP 应用

### 单元测试

```typescript
// tests/tools.test.ts
import { describe, it, expect, beforeEach } from 'vitest';
import { createUser, userTools } from '../src/tools/userTools.js';

describe('User Tools', () => {
  describe('createUser', () => {
    it('should create a user with required fields', async () => {
      const result = await createUser({
        name: 'John Doe',
        email: 'john@example.com'
      });

      expect(result.success).toBe(true);
      expect(result.userId).toMatch(/^usr_/);

        email: 'jane@example.com',

        role: 'admin'
      });

      expect(result.success).toBe(true);
      expect(result.data.age).toBe(30);
      expect(result.data.role).toBe('admin');
    });

    it('should throw on invalid email', async () => {
      await expect(
        createUser({

          email: 'invalid-email'
```

```
┌─────────────────────────────────────────────────────────────────┐
│              传统 AI 集成的"接口混乱"                             │
```

## MCP 的三大核心概念

就像 USB-C 有"数据传输"、"供电"、"视频输出"等功能，MCP 也有三大核心概念：

### 1. 工具（Tools）

**类比**：USB-C 的"数据传输"功能

工具允许 AI 执行具体的操作，如调用 API、执行代码、访问数据库等。

```python
# 定义一个"搜索文件"工具
{
    "name": "search_files",
    "description": "在指定目录中搜索文件",
    "inputSchema": {
        "type": "object",
        "properties": {
            "directory": {"type": "string"},
            "pattern": {"type": "string"}
        }
    }
}
```

### 2. 资源（Resources）

**类比**：USB-C 的"视频输出"功能（单向展示）

资源允许 AI 读取数据，如文件内容、数据库记录、API 响应等。

```python
# 定义一个"项目配置文件"资源
{
    "uri": "config://project/settings",
    "name": "Project Settings",
    "description": "当前项目的配置信息",
    "mimeType": "application/json"
}
```

### 3. 提示模板（Prompts）

**类比**：USB-C 的"协议协商"功能（设备间沟通）

提示模板是可复用的 AI 提示，帮助用户快速完成常见任务。

```python
# 定义一个"代码审查"提示
{
    "name": "code_review",
    "description": "对提交的代码进行审查",
    "arguments": [
        {"name": "diff", "description": "代码差异"}
    ]
}
```

## MCP 的传输方式

就像 USB-C 可以通过不同线缆传输（有线、雷电、DisplayPort Alt Mode），MCP 也支持多种传输方式：

| 传输方式 | 说明 | 适用场景 |
|----------|------|----------|
| **Stdio** | 标准输入输出 | 本地进程、本地开发 |
| **HTTP + SSE** | HTTP 请求 + Server-Sent Events | 远程服务、实时更新 |
| **Streamable HTTP** | 流式 HTTP | 大响应、高并发 |

## 一个完整的 MCP 示例

让我们用一个实际的例子来展示 MCP 的工作流程：

### 场景：让 AI 帮你搜索 GitHub 仓库

**1. 配置 MCP Server（相当于"连接设备"）**

```json
// claude_desktop_config.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxx"
      }
    }
  }
}
```

**2. Server 提供工具（相当于"设备能力列表"）**

```json
{
  "tools": [
    {
      "name": "search_repositories",
      "description": "搜索 GitHub 仓库"
    },
    {
      "name": "get_pull_request",
      "description": "获取 PR 详情"
    },
    {
      "name": "create_issue",
      "description": "创建 Issue"
    }
  ]
}
```

**3. AI 调用工具（相当于"使用设备功能"）**

```
你：帮我搜索一下有哪些关于 MCP 的热门仓库

AI（通过 MCP）：
  1. 调用 search_repositories 工具
  2. 解析返回结果
  3. 展示给你
```

## MCP vs USB-C 对比表

| USB-C | MCP | 说明 |
|-------|-----|------|
| 物理接口 | 协议接口 | 连接的"形式" |
| 设备能力 | 工具/资源 | 可以执行的操作 |
| 即插即用 | 动态发现 | 自动识别能力 |
| 供电 | 认证/授权 | 身份验证 |
| 高速传输 | JSON-RPC | 数据交换格式 |

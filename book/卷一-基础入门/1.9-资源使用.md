# 1.9 资源的定义与使用

> 理解 MCP 资源的概念和使用方法

---

## 什么是资源（Resources）

资源是 MCP 中用于**暴露数据**的概念，AI 可以读取这些数据来获取上下文。

```
┌─────────────────────────────────────────────────────────────────┐
│                         资源的特点                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✓ 只读操作——不会修改系统状态                                    │
│  ✓ 按需获取——AI 需要时再读取                                    │
│  ✓ 格式灵活——支持任意数据类型                                    │
│  ✓ URI 标识——通过唯一 URI 访问                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 资源定义结构

```json
{
  "uri": "resource://identifier",
  "name": "Resource Name",
  "description": "资源的描述",
  "mimeType": "application/json"
}
```

---

## 资源示例

```python
from fastmcp import FastMCP

app = FastMCP("File Resources")

@app.resource("config://app")
def get_app_config() -> dict:
    """返回应用配置"""
    return {
        "version": "1.0.0",
        "theme": "dark",
        "language": "zh-CN"
    }

@app.resource("file://readme")
def get_readme() -> str:
    """返回 README 内容"""
    with open("README.md", "r") as f:
        return f.read()
```

## 资源 vs 工具

| 特性 | 资源 | 工具 |
|------|------|------|
| 操作类型 | 读取 | 执行 |
| 系统状态 | 不变 | 可能改变 |
| 调用方式 | AI 主动读取 | AI 决定调用 |
| 返回类型 | 数据 | 执行结果 |

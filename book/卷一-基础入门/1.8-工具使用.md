# 1.8 工具的定义与使用

> 理解 MCP 工具的核心概念和使用方法

---

## 什么是工具（Tools）

工具是 MCP 的核心概念之一，允许 AI 执行具体的操作。

```
┌─────────────────────────────────────────────────────────────────┐
│                         工具 vs 资源                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  工具 (Tool)                 资源 (Resource)                     │
│  ├─ 可执行操作              ├─ 可读取数据                       │
│  ├─ 会改变系统状态          ├─ 系统状态不变                     │
│  ├─ 需要参数                ├─ 可选参数                         │
│  └─ 返回执行结果            └─ 返回数据内容                     │
│                                                                 │
│  示例：                     示例：                               │
│  • 发送邮件                 • 读取文件                          │
│  • 创建数据库记录           • 查询数据库                        │
│  • 调用 GitHub API          • 获取项目信息                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 工具定义结构

```json
{
  "name": "tool_name",
  "description": "工具的描述，AI 会根据这个决定是否使用",
  "inputSchema": {
    "type": "object",
    "properties": {
      "param_name": {
        "type": "string",
        "description": "参数的描述"
      }
    },
    "required": ["param_name"]
  }
}
```

## 工具示例：天气查询

```python
from fastmcp import FastMCP

app = FastMCP("Weather Server")

@app.tool()
def get_weather(city: str, unit: str = "celsius") -> str:
    """
    获取指定城市的天气信息

    Args:
        city: 城市名称，如"北京"
        unit: 温度单位，"celsius"或"fahrenheit"

    Returns:
        天气信息描述
    """
    # 模拟天气查询
    return f"{city}当前天气晴朗，温度 22°C"
```

---

## 最佳实践

1. **描述要清晰**——让 AI 知道何时该使用
2. **参数要有默认值**——提高易用性
3. **错误要处理**——返回有意义的错误信息
4. **返回要结构化**——便于 AI 解析

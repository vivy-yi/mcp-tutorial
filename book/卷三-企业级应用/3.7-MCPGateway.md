## 一、引子：微服务时代的网关

### 1.1 没有网关的困境

```
┌─────────────────────────────────────────────────────────────────┐
│                    无网关的问题                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  问题场景：                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 多个 MCP 服务需要单独管理                           │   │
│  │  • 每个服务都需要认证、限流                            │   │
│  │  • 客户端需要知道所有服务的地址                       │   │
│  │  • 跨服务调用难以追踪                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  痛点：                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  ⚠️ 客户端配置复杂                                    │   │
│  │  ⚠️ 安全策略分散，难以统一                            │   │
│  │  ⚠️ 无法跨服务追踪问题                                │   │
│  │  ⚠️ 扩展困难                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 网关的价值

```mermaid
flowchart LR
    subgraph "有网关后"
        A1[客户端] --> B1[MCP Gateway]
        B1 --> C1[认证]
        B1 --> D1[限流]
        C1 --> E1[路由到服务 A]
        C1 --> F1[路由到服务 B]
        C1 --> G1[路由到服务 C]
    end
```

## 三、架构设计：网关架构

### 3.1 网关架构图

```mermaid
flowchart TB
    subgraph "MCP Gateway 架构"
        A1[客户端] --> B1[负载均衡器]
        B1 --> C1[网关集群]

        C1 --> D1[认证模块]
        C1 --> D2[限流模块]
        C1 --> D3[路由模块]

        D1 --> E1[MCP 服务 A]
        D1 --> E2[MCP 服务 B]
        D1 --> E3[MCP 服务 C]

        C1 --> F1[日志收集]
        C1 --> F2[监控]
    end
```

### 3.2 请求处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    网关请求处理流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 负载均衡                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  选择一个网关节点处理请求                              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  2. 认证验证                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 验证 JWT Token                                      │   │
│  │  • 提取用户/租户信息                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  3. 限流检查                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 检查用户/租户配额                                  │   │
│  │  • 检查全局限流                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  4. 路由决策                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 根据工具名称确定目标服务                           │   │
│  │  • 选择健康的后端实例                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  5. 转发请求                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 转换协议（如需要）                                 │   │
│  │  • 添加跟踪头信息                                      │   │
│  │  • 转发到后端服务                                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  6. 返回响应                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 记录响应时间和状态                                │   │
│  │  • 返回给客户端                                      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、实践指南：部署与运维

```

│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │    [客户端]                                             │   │
│  │        │                                               │   │
│  │    [负载均衡器]                                        │   │
│  │        │                                               │   │
│  │  ┌───┴───┐                                           │   │
│  │  │       │                                           │   │
│  │ [GW1] [GW2] [GW3]                                    │   │
│  │  │       │                                           │   │
│  │  └───┬───┘                                           │   │
│  │        │                                               │   │

│  ┌─────────────────────────────────────────────────────────┐   │

│  │  • 请求成功率                                         │   │
│  │  • 限流触发次数                                      │   │
│  │  • 后端服务健康状态                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
```

## 二、核心概念：规范对比与映射

### 2.1 MCP 与 OpenAPI 的对应关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    MCP ↔ OpenAPI 映射                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  MCP 工具定义：                                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  {                                                      │   │
│  │    "name": "get_user",                                │   │
│  │    "description": "获取用户信息",                     │   │
│  │    "inputSchema": {                                   │   │
│  │      "type": "object",                                │   │
│  │      "properties": {                                   │   │
│  │        "user_id": {"type": "string"}                 │   │
│  │      }                                                │   │
│  │    }                                                  │   │
│  │  }                                                      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  OpenAPI 操作：                                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  paths:                                               │   │
│  │    /users/{user_id}:                                  │   │
│  │      get:                                              │   │
│  │        operationId: getUser                          │   │
│  │        description: 获取用户信息                       │   │
│  │        parameters:                                    │   │
│  │          - name: user_id                             │   │
│  │            in: path                                   │   │
│  │            schema:                                    │   │
│  │              type: string                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 映射规则详解

```mermaid
flowchart TB
    subgraph "MCP → OpenAPI"
        A1[工具名称] --> B1[operationId]
        A1 --> C1[路径模板]
        A2[描述] --> B2[description]
        A3[inputSchema] --> B3[parameters + requestBody]
    end

    subgraph "OpenAPI → MCP"
        D1[operationId] --> E1[工具名称]
        D1[路径] --> E2[路径模板]
        D3[parameters] --> E3[inputSchema]
    end
```

## 三、架构设计：自动生成系统

### 3.1 系统架构

```mermaid
flowchart TB
    subgraph "双向生成系统"
        A1[输入] --> B1[解析器]
        B1 --> C1[映射引擎]

        C1 --> D1[代码生成器]
        C1 --> E1[文档生成器]

        D1 --> F1[Python SDK]
        D1 --> G1[TypeScript SDK]
        D1 --> H1[Go SDK]

        E1 --> I1[Markdown 文档]
        E1 --> J1[交互式文档]
    end
```

### 3.2 生成流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    OpenAPI 生成流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 注册 MCP 工具                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  @app.tool()                                           │   │
│  │  def get_user(user_id: str) -> dict:                  │   │
│  │      """获取用户信息"""                                │   │
│  │      ...                                               │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  2. 解析工具定义                                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 提取名称: get_user                                  │   │
│  │  • 提取描述: 获取用户信息                               │   │
│  │  • 提取参数: {user_id: string}                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  3. 转换为 OpenAPI                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  paths:                                               │   │
│  │    /users/{user_id}:                                  │   │
│  │      get:                                              │   │
│  │        operationId: getUser                          │   │
│  │        description: 获取用户信息                       │   │
│  │        parameters:                                    │   │
│  │          - name: user_id                             │   │
│  │            in: path                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                         │                                       │
│                         ▼                                       │
│  4. 生成产物                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • openapi.yaml                                       │   │
│  │  • SDK 代码                                           │   │
│  │  • API 文档                                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    SDK 生成支持                                       │

│  │  from my_api import Client                             │   │

│  │  user = client.users.get(user_id="123")               │   │

│                                                                 │
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    本章核心要点                                    │

│                                                                 │
│  4. 价值                                                        │
│     • 一次定义，多处生成                                        │
│     • 文档和代码保持同步                                        │
│     • 生态互通无障碍                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

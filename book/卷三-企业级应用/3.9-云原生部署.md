## 二、核心概念：Docker 容器化

### 2.1 Docker 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    Docker 核心概念                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  镜像 (Image):                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 应用及其依赖的模板                                  │   │
│  │  • 只读                                               │   │
│  │  • 分层结构                                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  容器 (Container):                                             │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 镜像的运行实例                                      │   │
│  │  • 隔离的运行环境                                     │   │
│  │  • 可创建、启动、停止、删除                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  Dockerfile:                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 构建镜像的脚本                                     │   │
│  │  • 定义基础镜像、依赖、启动命令等                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 MCP 服务 Dockerfile

```dockerfile
# 基础镜像
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "server.py"]
```

### 3.2 MCP 服务 K8s 部署

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mcp-github-server
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mcp-github-server
  template:
    metadata:
      labels:
        app: mcp-github-server
    spec:
      containers:
      - name: server
        image: mcp/github-server:latest
        ports:
        - containerPort: 8000
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"
          requests:
            memory: "256Mi"
            cpu: "250m"
        env:
        - name: GITHUB_TOKEN
          valueFrom:
            secretKeyRef:
              name: mcp-secrets
              key: github-token
---
apiVersion: v1
kind: Service
metadata:
  name: mcp-github-server
spec:
  selector:
    app: mcp-github-server
  ports:
  - port: 80
    targetPort: 8000
  type: ClusterIP
```

## 四、实践指南：部署配置

### 4.1 部署策略

```
┌─────────────────────────────────────────────────────────────────┐
│                    K8s 部署策略                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  滚动更新 (Rolling Update):                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 逐步替换旧版本                                      │   │
│  │  • 零停机部署                                         │   │
│  │  • 快速回滚                                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  蓝绿部署 (Blue-Green):                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 两套完全相同的环境                                 │   │
│  │  • 瞬间切换                                          │   │
│  │  • 需要双倍资源                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  金丝雀部署 (Canary):                                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  • 小比例流量到新版本                                 │   │
│  │  • 逐步扩大比例                                       │   │
│  │  • 风险可控                                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  推荐: 滚动更新（简单可靠）                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
